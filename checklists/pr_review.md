# 🔎 Pull Request Review Checklist

## 🛠️ Functionality
- [ ] Does the code work as expected?
- [ ] Are all edge cases handled?
- [ ] Does it meet the requirements of the task/issue?

## 🧹 Code Quality
- [ ] Is the code easy to understand?
- [ ] Are variable and function names descriptive?
- [ ] Is there redundant code that can be simplified?
- [ ] Are there any console logs or "TODOs" left?

## 🧪 Testing
- [ ] Are there unit or integration tests?
- [ ] Do the tests cover the new logic?
- [ ] Do all tests pass locally?

## 🛡️ Security & Performance
- [ ] Is there any sensitive data (API keys) committed?
- [ ] Are new dependencies necessary and secure?
- [ ] Will this change cause a significant performance hit?

## 📖 Documentation
- [ ] Are complex logic blocks commented?
- [ ] Is the README or internal docs updated if necessary?
